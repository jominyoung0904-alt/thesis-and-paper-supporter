# 논문 작성 서포터 — 기획 회의 확정 사항 (2026-07-10)

> 제품 기획 회의(2026-07-10)에서 확정된 모든 결정 사항. PRD/SPEC 작성의 단일 소스.

## 1. 제품 비전

- **제품명(가칭)**: 논문 작성 서포터
- **한 줄 정의**: 대학원생의 논문 작성 전 과정을 "연구 프로젝트 메모리" 하나로 관통하며 보조하는 로컬 데스크톱 앱
- **타깃 사용자**: 한국 대학원생 (사회과학·공학 등 분야 무관), **컴퓨터 비숙련자** — 조금만 복잡해도 포기하는 사용자를 기준으로 UX 설계
- **논문 언어**: 국문 + 영문 모두 지원
- **차별화 포인트** (기존 도구 Elicit/SciSpace/scite/Zotero가 못 하는 것):
  1. 연구 전 과정을 관통하는 프로젝트 메모리 (파편화된 도구들의 빈자리)
  2. 연구 결정 이력 추적 (연구노트·심사 대응 자료로 활용)
  3. 인용 무결성 검증을 기본 내장한 집필 보조 (가짜 인용 구조적 차단)

## 2. 범위

- **아키텍처는 전 과정 통합**(주제선정→문헌조사→연구설계→데이터→집필→검토→심사대응)을 목표로 설계하되, **구현은 단계적**.
- **1단계 (이번 SPEC 범위)**:
  1. **연구 프로젝트 메모리 코어** — 연구 질문·가설·용어 정의·연구 결정 이력(왜 이 방법론/왜 이 문헌 제외)·지도교수 피드백을 구조화 저장. 모든 기능이 이 메모리를 공유 컨텍스트로 사용.
  2. **선행연구 딥리서치** — 사용자가 채팅으로 "이런 연구 있어?" 질문 → LLM이 국문+영문 검색어 생성 → KCI·ScienceON·Semantic Scholar API 병렬 메타데이터(제목·초록·저자·인용수) 검색 → 관련도 스크리닝 → 실제 링크 첨부된 종합 리포트. 학술 DB 직접 조회라 가짜 인용 구조적 불가.
  3. **집필·검토 보조** — 섹션별 품질 게이트 체크리스트(예: 서론 = 연구 갭 명시 + 기여 명시 + 모든 주장에 인용), 학술 문장 다듬기, 모의 심사(Reviewer 2 시뮬레이션 — 적대적 리뷰).
- **2단계 이후 (SPEC 범위 외, 아키텍처만 고려)**: PDF 라이브러리 RAG, 문헌 스크리닝 파이프라인, Zotero 연동.
- **UX 원칙**: 채팅 하나로 모든 기능 접근, 설정 화면 최소화, 모든 안내·에러는 한국어 일상어.

## 3. 배포 형태 (확정)

- **Electron 데스크톱 앱, 포터블 zip 배포** (Tauri 대신 Electron — WebView2 의존성 없이 모든 Win10/11에서 동작 보장, zip 150~250MB 허용)
- 설치 프로그램 없음, 레지스트리·AppData 미사용, 관리자 권한 불필요. 압축 해제 후 exe 더블클릭으로 즉시 실행.
- **폴더 구조**:
  ```
  논문서포터/
  ├── 논문서포터.exe
  ├── 처음이라면_읽어주세요.html   ← SmartScreen 대처 스크린샷 포함
  ├── app/                        ← 프로그램 본체 (업데이트 시 교체)
  ├── data/                       ← 사용자 데이터 (메모리, 라이브러리, 키)
  └── config/settings.json        ← 메모장으로도 수정 가능한 설정
  ```
- 원칙: "폴더 복사 = 백업", "폴더 이동 = 그대로 동작", "새 버전 = app/만 교체, data/ 보존"

## 4. LLM 연동 (확정)

- **멀티 프로바이더**: Claude / Gemini / OpenAI — LLM 호출부를 어댑터 계층으로 추상화
- **최초 실행 설정 마법사**에서 제공자 선택 + API 키 입력. 키는 로컬 암호화 저장, 설정에서 언제든 교체 가능.
- **무료/유료 모드 구분**:
  - 무료 모드 = Gemini Flash 무료 티어 (키 발급 마법사: Google AI Studio, 카드 불필요). 분당 10~15회 제한 → 앱에 rate limiter 내장, 딥리서치 1회 2~3분 안내.
  - 유료 모드 = 작업 복잡도별 모델 자동 배치 (스크리닝=경량 모델, 리포트 종합·모의 심사=상위 모델)
- **기능별 품질·프라이버시 안내** (하드코딩 금지, 원격 설정에서 로드):
  - 무료 모드: "결과 품질은 같지만 속도가 느리고, 입력 내용이 구글 AI 학습에 쓰일 수 있어 작성 중인 원고 업로드는 권하지 않아요"
  - 유료 모드: "Pro급 모델로 리포트·문장 다듬기·모의 심사 품질 향상 + 속도 제한 없음 + 입력 내용 학습 미사용"
  - 각 기능 화면에 현재 모드 배지 표시
- **월 요금 안내** (원격 설정으로 갱신, 2026-07 기준): Gemini Flash 절약형 월 ~$2 / Claude Haiku ~$5 / OpenAI mini ~$4 / 균형형 $8~15. 월 사용액 실시간 표시 + 상한 알림 기능 포함.

## 5. 학술 검색 API (확정 — 하이브리드)

- **대상**: KCI OPEN API(국내 학술지, 공공데이터포털 경유), ScienceON API(KISTI — 국내논문+학위논문, 토큰 2시간 만료 자동 갱신 필요), Semantic Scholar(해외, 무료)
- **하이브리드 키 전략**:
  1. 기본: **내장 공용 키**로 압축 해제 직후 즉시 검색 가능 (첫 경험 보장)
  2. 설정에 "내 학술 키 등록" 마법사 — 단계별 스크린샷 안내. KCI(공공데이터포털, 자동승인 가능성 높음) 1순위, ScienceON(수동 승인, 수일 소요)은 "선택 항목"으로 안내
  3. 공용 키 사용량 임박 시 개인 키 발급 유도 배너 ("5분 소요 업그레이드" 프레이밍)
  4. 내장 키는 원격 설정으로 교체·폐기 가능하게
- 원문(PDF) 접근은 불가 — 초록 기반 판단 + 원문 링크 제공 ("학교 도서관 계정으로 열람")

## 6. 서버 주소·원격 설정 (확정 — 이중화)

- **로컬**: config/settings.json에 모든 외부 주소(LLM 3사, KCI, ScienceON, Semantic Scholar) + 앱 내 설정 화면에서 수정 + "기본값 복원" 버튼
- **원격**: GitHub Pages 정적 호스팅에 endpoints.json — 앱 시작 시 확인, 실패 시 로컬 값 사용. 원격 설정으로 관리하는 것: API 주소, 모델명(단종 대응), 요금 안내표, 공지사항(새 버전 알림), 내장 학술 키 교체

## 7. 리스크 대응 11개 (확정 — 1~3번은 MVP 필수)

1. **SmartScreen 경고** — 안내 HTML에 "추가 정보 → 실행" 스크린샷 필수, 백신 오탐 안내 병행
2. **zip 미해제 실행 감지** — 앱이 자기 경로 검사, zip 내부/임시폴더면 안내 후 종료
3. **API 한도·키 오류 한국어 안내** — 429/키오류/크레딧 소진을 일상어로 번역 ("오늘 무료 사용량을 다 썼어요. 내일 오후 5시에 다시 채워져요") + 자동 재시도
4. **키 포함 폴더 공유 방지** — 키 파일 분리 + "다른 사람에게 줄 때는 키 제외 내보내기" 기능
5. **OneDrive/바탕화면 배치 감지** — 경로 감지 후 권장 위치 안내 (강제 아님)
6. **쓰기 권한 없는 위치** — 시작 시 쓰기 테스트, 실패 시 이동 안내
7. **기관 방화벽·프록시** — 시작 시 연결 진단 화면 (서비스별 ✅/❌) + 프록시 설정란
8. **내장 학술 키 공유 한도** — 호출량 모니터링, 원격 설정으로 키 교체 (5절 하이브리드로 완화)
9. **모델 단종** — 모델명 원격 설정 관리
10. **데이터 유실·버전 업데이트** — data/ 분리 + 주기적 자동 백업 + 데이터 내보내기
11. **장시간 작업 중단** — 딥리서치 체크포인트 저장 후 이어하기

## 8. 적용할 설계 패턴 (지식 위키에서 채택)

| 패턴 | 적용 |
|---|---|
| 검증 가능 AI / LLM 역할 제한 | LLM은 해석·요약만, 인용·서지정보는 학술 API로 결정론적 검증. 인용 클릭 → 원본 링크 |
| Lore식 결정 추적 | 연구 결정 기록(왜 이 방법론, 왜 이 문헌 제외) → 연구노트·심사 대응 자료 |
| AX / 조직 메모리 | "매 세션이 첫 날" 문제 대응 — 프로젝트 메모리를 모든 LLM 호출의 공유 컨텍스트로 주입 |
| 목표 주도 + 품질 게이트 | 섹션별 성공 기준 체크리스트 (절차 지시 대신 기준 충족 검증) |
| 다중 모델 Debate | 모의 심사(Reviewer 2)에 적대적 리뷰 전략 |
| 모델 라우팅 + 캐싱 | 작업 복잡도별 모델 배치, 프로젝트 메모리는 캐시 친화적 프리픽스로 설계 |
| 체크포인트 + 자가 치유 | 장시간 딥리서치 중단·재개 |
