# SPEC-TSA-002 수락 기준

## 시나리오

### S1: 새 프로젝트 생성
- Given: 사용자가 앱을 실행하고 프로젝트 전환 UI를 열었다.
- When: "새 프로젝트 만들기"를 선택하고 이름("2주차 논문")을 입력한다.
- Then: `data/projects/{새 projectId}/` 디렉터리가 생성되고, 프로젝트 목록에 새 항목이 추가되며, 즉시 해당 프로젝트로 전환된다.

### S2: 프로젝트 전환 및 데이터 격리
- Given: 프로젝트 A(연구질문 X 보유)와 프로젝트 B(연구질문 Y 보유)가 모두 존재한다.
- When: 사용자가 프로젝트 A에서 B로 전환한다.
- Then: 채팅/보관함/리서치이력/점검기록 화면 전부가 B의 데이터만 표시하며, A의 연구질문 X나 A의 대화 내역이 어디에도 노출되지 않는다.

### S3: 기존 default 프로젝트 자동 마이그레이션
- Given: Sprint 1 시절 `data/projects/default/memory.json`이 이미 존재하는 상태에서 앱을 이번 스프린트 반영본으로 최초 실행한다.
- When: 앱이 부트스트랩된다.
- Then: 해당 데이터가 "내 연구 1" 프로젝트로 자동 편입되어 프로젝트 목록에 나타나고, 앱을 재시작해도 동일 마이그레이션이 중복 실행되지 않는다.

### S4: 프로젝트 이름 변경
- Given: 활성 프로젝트가 존재한다.
- When: 사용자가 이름을 "졸업논문(최종)"으로 변경한다.
- Then: 프로젝트 전환 UI와 모든 화면 헤더에 변경된 이름이 즉시 반영되고, 데이터 파일 경로(projectId)는 변하지 않는다.

### S5: 프로젝트 삭제(보관)
- Given: 프로젝트 C가 존재하며 활성 프로젝트가 아니다.
- When: 사용자가 C를 삭제한다.
- Then: C는 프로젝트 전환 목록에서 사라지지만 `data/projects/{C의 id}/` 데이터는 디스크에 그대로 남아 있다(archived: true).

### S6: 문헌 저장 → 보관함
- Given: 딥리서치 결과 화면에 참고문헌 5편이 표시되어 있다.
- When: 사용자가 그 중 2편에 저장 버튼(체크)을 누른다.
- Then: 현재 프로젝트의 보관함에 해당 2편의 PaperMetadata 전체, 저장일, 출처 리서치 식별자가 저장되고, 보관함 화면에서 즉시 조회된다.

### S7: 보관함 → 서지 복사
- Given: 보관함에 국문 문헌 1편과 영문 문헌 1편이 저장되어 있다.
- When: 사용자가 두 문헌을 모두 선택하고 "서지 복사"를 누른다.
- Then: 클립보드에 APA 형식 텍스트 2줄이 복사되며, 국문 문헌은 국문 표기 규칙, 영문 문헌은 영문 표기 규칙(et al. 등)이 각각 적용된다.

### S8: 리서치 이력 재열람 → 회의하기
- Given: 지난 주 실행한 딥리서치 결과가 리서치 이력에 저장되어 있다.
- When: 사용자가 이력 목록에서 해당 결과를 열고 "이 결과로 회의하기"를 누른다.
- Then: 채팅 모드로 전환되며 새 대화가 시작되고, 첫 턴(summary)에 해당 리포트 요약과 참고문헌 목록이 포함되어 있다.

### S9: 대화 자동 저장 → 이어하기
- Given: 사용자가 채팅에서 3턴을 주고받았다(앱을 종료하지 않음).
- When: 사용자가 대화 목록 화면을 열었다가 다시 원래 대화를 선택한다.
- Then: 3턴 전체가 그대로 복원되어 이어서 대화할 수 있다. 별도 "저장" 버튼을 누른 적이 없어도 데이터가 보존되어 있다.

### S10: 새 대화 시작
- Given: 사용자가 기존 대화를 이어가는 중이다.
- When: "새 대화 시작" 버튼을 누른다.
- Then: 빈 대화 세션이 시작되고, 기존 대화는 대화 목록에 그대로 남아 언제든 다시 열 수 있다.

### S11: 서론 점검 기록 저장 및 재열람
- Given: 사용자가 서론 초안으로 품질 게이트를 1회 실행했다.
- When: 점검 기록 이력 화면을 연다.
- Then: 실행 당시의 원고 텍스트, GateResult(통과/실패 항목별 피드백), 실행 시각이 표시된다.

### S12: 자동 백업 생성 및 최근 5개 보관
- Given: 사용자가 세션 동안 여러 프로젝트에 데이터를 남긴 채 앱을 종료한다.
- When: 앱 종료 처리가 실행된다.
- Then: `data/` 전체를 담은 백업 zip이 생성되고, 이미 5개의 백업이 있었다면 가장 오래된 것이 삭제되어 총 5개만 유지된다.

### S13: 딥리서치 체크포인트 재개
- Given: 딥리서치가 스크리닝 단계까지 진행된 상태에서 앱이 중단되었다(체크포인트 저장됨).
- When: 사용자가 같은 프로젝트에서 앱을 재실행하고 동일 리서치를 다시 시작한다.
- Then: 검색어 생성·조회 단계를 반복하지 않고 마지막 체크포인트(스크리닝 진행률)부터 이어서 진행한다.
