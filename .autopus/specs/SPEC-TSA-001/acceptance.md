# SPEC-TSA-001 수락 기준

## 시나리오

### S1: zip 압축 해제 후 첫 실행 플로우 (NFR-DEP-001, NFR-DEP-002, NFR-DEP-004, NFR-LLM-002)
- Given: 사용자가 논문서포터.zip을 다운로드했고 아직 압축을 해제하지 않았다.
- When: 사용자가 zip 파일을 더블클릭해 압축 해제 없이 내부의 논문서포터.exe를 바로 실행하려 한다.
- Then: 앱은 자신이 zip 내부/임시 폴더에서 실행 중임을 감지하고, 압축을 먼저 해제하라는 한국어 안내 메시지를 표시한 후 종료한다.

- Given: 사용자가 zip을 바탕화면에 압축 해제했다.
- When: 사용자가 논문서포터.exe를 더블클릭하고 SmartScreen 경고가 뜬다.
- Then: 처음이라면_읽어주세요.html에 있는 "추가 정보 > 실행" 스크린샷 안내를 따라 실행하면, 앱이 정상 구동되고 최초 실행 설정 마법사(프로바이더 선택, API 키 입력)가 표시된다.

### S2: 무료 모드 rate limit 하 딥리서치 완주 (FR-RES-001~006, NFR-LLM-003)
- Given: 사용자가 설정 마법사에서 무료 모드(Gemini Flash)를 선택하고 API 키를 등록했다.
- When: 사용자가 채팅창에 "국내 대학생 SNS 중독 관련 연구 있어?"를 입력한다.
- Then: 앱은 분당 10~15회 호출 제한을 준수하며 국문/영문 검색어를 생성하고, OpenAlex/Semantic Scholar(및 개인 키 등록 시 KCI/ScienceON)를 병렬 조회한 뒤, 2~3분 이내에 실제 링크가 첨부된 종합 리포트를 반환하며 rate limit으로 인한 지연 시 "속도 제한으로 시간이 걸리고 있어요" 등 진행 상황을 안내한다.

### S3: 한국어 에러 메시지 (NFR-LLM-004)
- Given: 사용자가 무료 모드로 딥리서치를 실행 중이며 당일 무료 사용량을 모두 소진했다.
- When: LLM 프로바이더가 429(rate limit exceeded) 오류를 반환한다.
- Then: 앱은 원본 오류 코드/영문 메시지를 사용자에게 노출하지 않고 "오늘 무료 사용량을 다 썼어요. 내일 오후 5시에 다시 채워져요"와 같은 한국어 일상어 안내를 표시하며, 정책상 재시도 가능한 경우 자동으로 재시도한다.

- Given: 사용자가 등록한 API 키가 잘못되었거나 만료되었다.
- When: LLM 프로바이더가 인증 오류(401/403 등)를 반환한다.
- Then: 앱은 "입력하신 키가 올바르지 않은 것 같아요. 설정에서 키를 다시 확인해 주세요" 등 한국어 안내를 표시하고 설정 화면으로 이동할 수 있는 버튼을 제공한다.

### S4: 키 제외 내보내기 (NFR-RISK-004)
- Given: 사용자가 다른 사람에게 프로젝트 폴더를 공유하려 한다.
- When: 사용자가 설정에서 "키 제외 내보내기"를 실행한다.
- Then: 앱은 API 키, 학술 키 등 인증 정보가 담긴 파일을 제외한 프로젝트 메모리/리서치 결과만 별도 폴더 또는 압축 파일로 생성하며, 내보낸 결과물에 키 파일이 포함되지 않았음을 검증할 수 있다.

### S5: 체크포인트 재개 (FR-RES-007, FR-RES-008)
- Given: 사용자가 딥리서치를 실행 중이며 검색어 생성과 OpenAlex 조회까지 완료되고 Semantic Scholar 조회가 진행 중이다.
- When: 네트워크 단절 또는 앱 강제 종료로 딥리서치가 중단된다.
- Then: 앱은 중단 직전까지의 진행 상태(생성된 검색어, 완료된 API 조회 결과)를 체크포인트로 저장해 두었으며, 사용자가 앱을 재실행하면 처음부터 다시 조회하지 않고 마지막 체크포인트 이후 단계(예: 남은 API 조회 또는 스크리닝)부터 이어서 진행한다.

### S6: 프로젝트 메모리 공유 컨텍스트 주입 (FR-MEM-001~003)
- Given: 사용자가 프로젝트 메모리에 연구 질문과 가설을 등록했다.
- When: 사용자가 딥리서치 또는 집필 보조 기능을 사용하기 위해 LLM을 호출한다.
- Then: 해당 LLM 호출의 프롬프트 컨텍스트에 등록된 연구 질문과 가설이 자동으로 포함되어, 사용자가 매번 배경을 다시 설명하지 않아도 일관된 응답을 받는다.

### S7: 인용 무결성 보장 (FR-RES-005)
- Given: 딥리서치 리포트가 생성되었다.
- When: 리포트에 포함된 인용 항목을 코드 수준에서 검증한다.
- Then: 모든 인용 항목의 서지정보(저자, 제목, 발행연도)는 학술 API(OpenAlex/Semantic Scholar 등) 응답의 ID와 1:1로 매핑되며, LLM 생성 텍스트에서 직접 파생된 서지정보 필드는 존재하지 않는다.

### S8: 품질 게이트에 의한 완료 차단 (FR-WRT-001, FR-WRT-002)
- Given: 사용자가 서론 섹션을 작성 중이며 연구 갭 명시, 기여 명시, 인용 여부 중 하나 이상을 충족하지 못했다.
- When: 사용자가 서론 섹션을 "완료"로 표시하려 시도한다.
- Then: 앱은 완료 처리를 차단하거나 미충족 항목을 명확히 표시하는 경고를 띄우며, 모든 기준을 충족한 이후에만 완료 처리를 허용한다.

### S9: 학술 검색 API 부분 실패 처리 (FR-RES-009)
- Given: 사용자가 딥리서치를 요청했고 Semantic Scholar는 결과를 반환했으나 OpenAlex는 관련 문헌을 찾지 못했다.
- When: 리포트가 생성된다.
- Then: 리포트에는 "국내외 통합 DB(OpenAlex)에서는 관련 문헌을 찾지 못했어요"라는 안내와 함께 Semantic Scholar 결과만으로 구성된 리포트가 제공되며, 국내 문헌이 있는 것처럼 오인되지 않는다.

### S10: 원격 설정 폴백 (NFR-CFG-002)
- Given: 앱이 시작되며 GitHub Pages의 endpoints.json 조회를 시도한다.
- When: 네트워크 문제로 원격 설정 조회가 실패한다.
- Then: 앱은 로컬 config/settings.json의 값으로 폴백하여 정상 동작하며, 앱 실행 자체가 중단되지 않는다.
