# 원격 설정(endpoints.json) 업데이트 방법

이 폴더의 `endpoints.json`은 앱이 실행될 때마다 자동으로 확인하는 "원격 설정 파일"이에요.
이 파일을 GitHub 웹사이트에서 수정하고 저장(커밋)하면, 몇 분 안에 **이미 설치되어 있는 모든 사용자의 앱**에도
새 설정이 자동으로 반영됩니다. 별도로 새 버전을 배포하거나 사용자가 다시 설치할 필요가 없어요.

개발 지식이 없어도 아래 3단계만 따라 하시면 됩니다.

## 1단계: GitHub 웹사이트에서 파일 열기

1. 웹브라우저로 다음 주소에 접속하세요.
   `https://github.com/jominyoung0904-alt/thesis-and-paper-supporter`
2. 파일 목록에서 `remote-config` 폴더 → `endpoints.json` 파일을 클릭하세요.
3. 화면 오른쪽 위의 연필(✏️) 모양 아이콘을 클릭하면 편집 모드로 바뀝니다.
   (GitHub 로그인이 되어 있지 않다면 먼저 로그인해 주세요.)

## 2단계: 필요한 값만 수정하기

가장 자주 하게 될 작업은 **AI 모델명이 단종되었을 때 새 모델명으로 바꾸는 것**이에요.
파일 안에서 `"models"` 부분을 찾아 원하는 값만 바꾸면 됩니다.

예시: OpenAI가 `gpt-5-mini` 모델을 없애고 `gpt-5.5-mini`를 새로 냈다면 —

수정 전:
```json
"models": {
  "gemini": "gemini-2.5-flash",
  "claude": "claude-sonnet-5",
  "openai": "gpt-5-mini"
}
```

수정 후 (openai 값만 변경):
```json
"models": {
  "gemini": "gemini-2.5-flash",
  "claude": "claude-sonnet-5",
  "openai": "gpt-5.5-mini"
}
```

### 지켜야 할 규칙

- 모델명은 영문자, 숫자, 그리고 `.` `_` `:` `-` 기호만 사용할 수 있어요(한글·공백·특수문자 불가).
- 모델명 길이는 1~100자까지만 허용돼요.
- 이 규칙을 벗어난 값은 앱이 자동으로 무시하고 기존 값을 그대로 사용하니, 잘못 입력해도 앱이 망가지지는 않아요.
- `endpoints`(API 주소) 값을 바꿀 때는 반드시 `https://`로 시작해야 하고, 각 서비스의 공식 도메인(예: `anthropic.com`,
  `googleapis.com`, `openai.com`)이어야 앱이 값을 받아들여요. 다른 도메인으로 바꾸면 무시됩니다(보안 보호 장치).

## 3단계: 저장(커밋)하기

1. 화면 아래쪽의 "Commit changes..." 버튼을 클릭하세요.
2. 커밋 메시지 칸에 무엇을 바꿨는지 간단히 한 줄로 적어주세요.
   예: `openai 모델명을 gpt-5.5-mini로 변경`
3. "Commit directly to the main branch" 선택된 상태에서 초록색 "Commit changes" 버튼을 누르면 끝이에요.

이제부터 앱을 실행하는 모든 사용자에게 몇 분 안에 새 설정이 자동으로 전달됩니다.
사용자가 인터넷에 연결되어 있지 않거나 서버 접속에 실패하면, 앱은 조용히 기존 로컬 설정으로 계속 동작해요
(앱이 멈추거나 오류를 내지 않습니다).
